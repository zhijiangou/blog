# 网站部署指南

## 概述

为了保持管理页面的发布功能，我们推荐使用 **Vercel + Supabase** 方案。

---

## 第一步：配置 Git 并推送到 GitHub

### 1. 配置 Git 用户信息

```bash
git config user.email "your-email@example.com"
git config user.name "Your Name"
```

### 2. 提交代码

```bash
git add .
git commit -m "Initial commit - Personal blog"
```

### 3. 在 GitHub 上创建新仓库

1. 访问 https://github.com/new
2. 创建一个新仓库（不要初始化 README）
3. 复制仓库的 HTTPS 或 SSH 地址

### 4. 推送到 GitHub

```bash
git remote add origin https://github.com/你的用户名/仓库名.git
git branch -M main
git push -u origin main
```

---

## 第二步：使用 Vercel 部署（推荐）

### 1. 访问 Vercel

打开 https://vercel.com 并使用 GitHub 账号登录

### 2. 导入项目

1. 点击 "Add New" → "Project"
2. 选择刚才推送到 GitHub 的仓库
3. 点击 "Import"

### 3. 配置项目（保持默认即可）

- Framework Preset: Next.js
- Root Directory: ./
- Build Command: 自动检测
- Output Directory: 自动检测

### 4. 点击 "Deploy"

等待 1-2 分钟，部署完成后您会获得一个网站地址（如 `https://your-blog.vercel.app`）

---

## 第三步：设置 Supabase（用于数据存储）

当前网站使用本地 JSON 文件存储数据，部署到 Vercel 后需要使用云数据库。

### 1. 创建 Supabase 项目

1. 访问 https://supabase.com
2. 注册/登录账号
3. 点击 "New Project"
4. 填写项目信息：
   - Name: 您的博客名称
   - Database Password: 设置一个安全密码（请记住！）
   - Region: 选择离您最近的区域
5. 点击 "Create new project"（等待约 2 分钟）

### 2. 获取连接信息

项目创建完成后：
1. 进入项目设置 → API
2. 复制以下信息：
   - Project URL
   - anon/public key

### 3. 创建数据库表

在 Supabase 的 SQL Editor 中执行以下 SQL：

```sql
-- 创建相册表
CREATE TABLE albums (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  description TEXT,
  cover_image TEXT,
  date DATE NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW())
);

-- 创建相册标签表
CREATE TABLE album_tags (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  album_id TEXT REFERENCES albums(id) ON DELETE CASCADE,
  tag TEXT NOT NULL
);

-- 创建照片表
CREATE TABLE photos (
  id TEXT PRIMARY KEY,
  album_id TEXT REFERENCES albums(id) ON DELETE CASCADE,
  title TEXT,
  url TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW())
);

-- 创建照片标签表
CREATE TABLE photo_tags (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  photo_id TEXT REFERENCES photos(id) ON DELETE CASCADE,
  tag TEXT NOT NULL
);

-- 创建文章/心情表
CREATE TABLE posts (
  id TEXT PRIMARY KEY,
  title TEXT,
  content TEXT NOT NULL,
  date DATE NOT NULL,
  is_mood BOOLEAN DEFAULT true,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW())
);

-- 创建音乐表
CREATE TABLE music (
  id TEXT PRIMARY KEY,
  title TEXT NOT NULL,
  artist TEXT NOT NULL,
  album TEXT,
  cover_url TEXT,
  url TEXT,
  platform TEXT DEFAULT 'netease',
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW())
);

-- 创建社交媒体表
CREATE TABLE social_links (
  id TEXT PRIMARY KEY,
  platform TEXT NOT NULL,
  title TEXT NOT NULL,
  url TEXT NOT NULL,
  thumbnail_url TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT TIMEZONE('utc'::text, NOW())
);

-- 启用行级安全（可选，开发阶段可以先禁用）
ALTER TABLE albums DISABLE ROW LEVEL SECURITY;
ALTER TABLE album_tags DISABLE ROW LEVEL SECURITY;
ALTER TABLE photos DISABLE ROW LEVEL SECURITY;
ALTER TABLE photo_tags DISABLE ROW LEVEL SECURITY;
ALTER TABLE posts DISABLE ROW LEVEL SECURITY;
ALTER TABLE music DISABLE ROW LEVEL SECURITY;
ALTER TABLE social_links DISABLE ROW LEVEL SECURITY;
```

### 4. 配置 Vercel 环境变量

在 Vercel 项目设置中：
1. 进入 Settings → Environment Variables
2. 添加以下变量：
   - `NEXT_PUBLIC_SUPABASE_URL`: 您的 Supabase Project URL
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`: 您的 Supabase anon key

### 5. 安装 Supabase 客户端

```bash
npm install @supabase/supabase-js
```

## 替代方案：如果不想改代码

如果您暂时不想修改代码使用云数据库，也可以：

### 方案 A：使用 Vercel + 定期导出 JSON

每次在管理后台修改后，手动下载 JSON 文件并提交到 GitHub，Vercel 会自动重新部署。

### 方案 B：使用 Netlify

类似 Vercel，Netlify 也支持 Next.js 和 Serverless Functions。

## 常见问题

### Q: 部署后管理页面不能保存数据？
A: 这是因为 Vercel 的文件系统是只读的。必须使用云数据库（如 Supabase、Firebase）或其他持久化方案。

### Q: 图片存储在哪里？
A: 可以使用 Supabase Storage、Firebase Storage 或 Cloudinary 等服务。

### Q: 可以使用 GitHub Pages 吗？
A: GitHub Pages 只支持静态网站，不支持 API 路由，所以管理页面无法使用。推荐使用 Vercel 或 Netlify。
